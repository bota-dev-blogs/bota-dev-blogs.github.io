# Pipeline 1 Layout Catalog

| Layout | Best use | Card count |
|---|---|---|
| `row` | equally weighted compact ideas | 1-3 |
| `timeline` | ordered stages without arrows | 2-4 |
| `spotlight` | one dominant idea with support | 1-4 |
| `stacked` | ordered rules or layers | 2-4 |
| `grid` | balanced evidence or concepts | 2-4 |
| `mosaic` | one large concept with smaller facets | 3-4 |
| `compare` | two alternatives plus optional synthesis | 2-3 |
| `lanes` | parallel tracks or categories | 2-4 |
| `checklist` | concrete requirements or tests | 2-4 |
| `flow-board` | process stages arranged by order | 2-4 |
| `bento` | one thesis and several compact details | 3-4 |
| `split-focus` | primary path versus constraints | 2-4 |
| `staircase` | progression, maturity, or escalation | 2-4 |
| `bands` | stacked layers or operating bands | 2-4 |
| `triptych` | three peer views plus optional synthesis | 3-4 |
| `dashboard` | headline metric or result with evidence | 3-4 |
| `orbit` | peer factors around one conceptual center | 3-4 |
| `matrix` | criteria, evidence, or paired dimensions | 2-4 |
| `system-board` | one control point with surrounding signals | 2-4 |
| `failure-focus` | one failure mode with causes or mitigations | 2-4 |
| `radial` | one central primitive with supporting routes or actors | 3-4 |
| `route-map` | an ordered journey with turns, recovery, or handoffs | 3-4 |
| `signal-board` | a continuous signal becoming several artifacts | 3-4 |
| `console` | one primary system with live status or supporting controls | 2-4 |

Do not select a layout randomly. Match visual hierarchy to the source argument,
and avoid using the same visual silhouette for adjacent GIFs unless the
relationship is genuinely the same. The renderer gives each layout its own
rail, scan, orbit, status, or hierarchy grammar; the storyboard should supply
the semantic content, not pixel coordinates.
